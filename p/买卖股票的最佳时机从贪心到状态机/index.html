买卖股票的最佳时机：从贪心到状态机

Feb 24, 2026
阅读时长: 6 分钟

本文将通过力扣（LeetCode）上经典的三道“买卖股票”问题（121, 122, 123），详细解析贪心算法以及**状态机（动态规划）**在其中的应用，并探讨这两种算法思想的本质区别。下面提供了完整的 Go 语言代码实现，包含了输入和输出测试用例以便于本地运行验证。

1. 力扣 121. 买卖股票的最佳时机（贪心）

题目描述：给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。你只能选择某一天买入这只股票，并选择在未来的某一个不同的日子卖出该股票。设计一个算法来计算你所能获取的最大利润。

贪心思路： 因为我们只能进行一次完整的交易（买入一次，卖出一次），想要利润最大化，我们需要在最低点买入，在之后的最高点卖出。 这道题的贪心策略在于：遍历价格数组时，我们只关心两件事，并在每一步都做出局部最优选择：

  1. 截止到今天，股票的历史最低价是多少？（贪心地寻找最低买入点 minP，越低越好）
  2. 如果今天卖出，利润是多少？（当前价格减去历史最低价），并与历史最大利润比较，贪心地更新最大利润 ans。

package main

import (
    "fmt"
)

// maxProfit121 核心逻辑
func maxProfit121(prices []int) (ans int) {
    if len(prices) == 0 {
        return 0
    }
    minP := prices[0]
    for _, p := range prices {
        ans = max(ans, p-minP)
        minP = min(minP, p)
    }
    return ans
}

func main() {
    prices := []int{7, 1, 5, 3, 6, 4}
    fmt.Printf("121题输入: prices = %v\n", prices)
    fmt.Printf("121题输出: 最大利润 = %d (预期: 5)\n", maxProfit121(prices))
}

2. 力扣 122. 买卖股票的最佳时机 II（贪心）

题目描述：给你一个整数数组 prices ，其中 prices[i] 表示某支股票第 i 天的价格。在每一天，你可以决定是否购买和/或出售股票。你在任何时候最多只能持有一股股票。你可以先购买，然后在同一天出售。返回你能获得的最大利润。

贪心思路： 这道题是最典型的贪心算法。因为交易次数不限，只要有钱赚我们就可以交易。 贪心策略：我们不关心跨度多长的买卖，只关注相邻两天的价格差。只要明天的价格比今天高（即 prices[i] - prices[i-1] > 0），我们就当作是今天买入，明天卖出。这就好比我们有上帝视角，只要发现接下来一天是涨的，我们就吃下这段利润。把所有上涨区间的利润全部收集起来，局部最优累加起来就是全局最优。

package main

import (
    "fmt"
)

// maxProfit122 核心逻辑
func maxProfit122(prices []int) (ans int) {
    for i := 1; i < len(prices); i++ {
        ans += max(0, prices[i]-prices[i-1])
    }
    return ans
}

func main() {
    prices := []int{7, 1, 5, 3, 6, 4}
    fmt.Printf("122题输入: prices = %v\n", prices)
    fmt.Printf("122题输出: 最大利润 = %d (预期: 7)\n", maxProfit122(prices))
}

3. 力扣 123. 买卖股票的最佳时机 III（状态机 / 动态规划）

题目描述：给定一个数组，它的第 i 个元素是一支给定的股票在第 i 天的价格。设计一个算法来计算你所能获取的最大利润。你最多可以完成 两笔 交易。注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

状态机思路（注意：这里不是单纯的贪心）： 由于限制了最多两笔交易，我们不能像 122 题那样贪心地收集所有上涨区间的利润。这道题本质上是状态机（动态规划）。 在每一天，我们的账户可能处于以下四个核心状态之一。我们需要定义变量来记录到达第 i 天时各个状态下的最大手头现金：

  • buy1: 第一次买入后的状态（花钱买入：选择继续保持之前的 buy1 状态，或者今天买入变为 -p）
  • sell1: 第一次卖出后的状态（赚钱卖出：选择继续保持之前的 sell1 状态，或者今天按 p 卖出变为 buy1 + p）
  • buy2: 第二次买入后的状态（花钱买入：选择继续保持之前的 buy2 状态，或者今天用 sell1 的钱买入变为 sell1 - p）
  • sell2: 第二次卖出后的状态（赚钱卖出：选择继续保持之前的 sell2 状态，或者今天按 p 卖出变为 buy2 + p）

每天我们都在根据前一天的状态，推导出今天的状态。这里的 max 操作是为了在相同状态之间做出最优选择（即动态规划的“最优子结构”），而不是像 122 题那样无脑累加利润。最终 sell2 积累的就是经过两次完整状态流转后的全局最大利润。

package main

import (
    "fmt"
    "math"
)

// maxProfit123 核心逻辑
func maxProfit123(prices []int) int {
    if len(prices) == 0 {
        return 0
    }
    // 初始状态，还没有买入时相当于花了无穷大的钱，没卖出时利润为0
    buy1, sell1, buy2, sell2 := math.MinInt32, 0, math.MinInt32, 0

    for _, p := range prices {
        buy1 = max(buy1, -p)
        sell1 = max(sell1, buy1+p)
        buy2 = max(buy2, sell1-p)
        sell2 = max(sell2, buy2+p)
    }
    return sell2
}

func main() {
    prices := []int{3, 3, 5, 0, 0, 3, 1, 4}
    fmt.Printf("123题输入: prices = %v\n", prices)
    fmt.Printf("123题输出: 最大利润 = %d (预期: 6)\n", maxProfit123(prices))
}

贪心算法与状态机（动态规划）的区别

通过上述三道题目的对比，我们可以清晰地看到这两种思想的差异：

1. 核心思想

  • 贪心算法 (Greedy)：只顾眼前，每一步都做出在当前看来最好的选择（局部最优）。它不考虑整体规划，也不记录复杂的历史状态，只希望通过一系列的局部最优操作，自然而然地累加出全局最优解。例如 122 题，只要明天比今天高，今天就买、明天就卖，无脑收集每一段上涨空间。
  • 状态机/动态规划 (State Machine / DP)：着眼全局，通过定义不同的状态，并根据状态转移方程，将一个复杂问题拆解为多个子问题。在每一个阶段，它会计算所有可能状态下的最优解，并记录下来供下一阶段使用。例如 123 题，它必须精确地记录“第一次买”、“第一次卖”、“第二次买”、“第二次卖”这四个独立且互相依赖的状态，无法用简单的“见涨就买”来解决。

2. 是否具有“后效性”

  • 贪心算法：做出的选择不能回退，一旦决定了就无法改变。它假设当前的选择不会影响未来的最优选择（无后效性）。
  • 状态机/动态规划：在计算某一个阶段的状态时，往往需要综合考虑前面各个阶段的状态。它通过维护一系列的状态表（或者压缩后的状态变量，如 buy1, sell1, buy2, sell2），保留了各种可能性的最优解，以便后续步骤做出全局最优决策。

3. 适用场景

  • 贪心算法：适用于问题具有“贪心选择性质”的场景，即局部最优解一定能推导出全局最优解。一旦题目加上了严格的限制（比如 123 题限制最多两次交易），贪心策略通常就会失效，因为当前的“局部最优”可能会吃掉未来更好的交易机会。
  • 状态机/动态规划：适用于存在重叠子问题和最优子结构，且有多种状态互相转换的复杂场景。只要能画出状态转移图，写出状态转移方程，就能用一套标准化的代码模版解决（如股票系列中限制交易 $k$ 次、含有冷冻期、含有手续费等进阶问题）。

总结

在处理“买卖股票”系列问题时，解题思路的演进过程就是算法思想升华的过程：

  1. 121题（单一限制）：维护单一边界极值，是最基础的贪心。
  2. 122题（无限制）：收集所有正收益，是贪心算法的完美体现。
  3. 123题（次数限制）：贪心失效，必须引入**状态机模型（动态规划）**来精确控制交易次数的流转和每次流转下的最大收益。

理解这三道题，就能打通从“见好就收（贪心）”到“统筹规划（状态机）”的算法任督二脉。
