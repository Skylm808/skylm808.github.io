贪心、动态规划、背包问题：一份讲清楚

这份笔记的目标不是堆题，而是把三件事讲明白：

  1. 贪心 和 动态规划 到底有什么区别
  2. 背包 为什么是题目模型，而不是一种算法
  3. 遇到题目时，怎么判断该往哪种思路上靠

1. 先分清：贪心、动态规划、背包分别是什么

这三个词经常一起出现，但它们不是一个层级。

1.1 贪心

贪心是一种解题思想：

  • 每一步都选当前看起来最优的方案
  • 不回头修改之前的决定

它成立的前提是：

局部最优可以推出全局最优

如果这个性质不成立，贪心就会错。

1.2 动态规划

动态规划也是一种解题思想：

  • 把大问题拆成小问题
  • 记录已经算过的状态
  • 通过状态转移得到全局最优

它常见的两个判断条件是：

  • 最优子结构
  • 重叠子问题

1.3 背包

背包不是算法，而是一种题目模型：

  • 有容量限制
  • 有一批物品
  • 每件物品有“代价”和“收益”或者“效果”
  • 问最优值 / 是否可行 / 最少数量 / 方案数

所以关系是：

  • 贪心、动态规划：解题方法
  • 背包：题目模型

面试里一句话总结：

贪心和动态规划是解题思想，背包是题目模型。背包题大多数用动态规划来做，因为它通常不满足贪心选择性质。

2. 贪心思想

2.1 贪心的本质

贪心的核心就一句话：

当前这一步怎么选最优，就直接这么选。

它的优点是：

  • 写法短
  • 复杂度低
  • 很适合线性扫描

它的难点也在这里：

  • 你必须能证明“局部最优 -> 全局最优”

所以贪心最难的不是写代码，而是证明为什么这样选不会错。

2.2 经典例题：LeetCode 121 买卖股票的最佳时机

题目本质

  • 只能买一次、卖一次
  • 问最大利润

为什么能贪心？

因为遍历到第 i 天时，你只关心两件事：

  • 之前出现过的最低价格
  • 今天卖出能得到的最大利润

局部最优就是：

  • 买入价越低越好
  • 当前利润越大越好

这个局部选择不会影响未来，所以贪心成立。

package main

func maxProfit(prices []int) int {
    minPrice := prices[0]
    ans := 0

    for i := 1; i < len(prices); i++ {
        if prices[i] < minPrice {
            minPrice = prices[i]
        }
        if prices[i]-minPrice > ans {
            ans = prices[i] - minPrice
        }
    }
    return ans
}

面试怎么说

这题贪心的关键是维护历史最低价。遍历到今天时，只要看“今天卖出”相对历史最低价的利润即可，因为买入越低越优，这个局部选择和未来不冲突。

2.3 进阶例题：LeetCode 45 跳跃游戏 II

题目本质

  • 数组 nums[i] 表示从位置 i 最多能跳多远
  • 问从下标 0 跳到最后一个位置的最少跳跃次数

贪心思路

每走到一层区间时：

  • 统计这一层能扩展到的最远位置 farthest
  • 当遍历到当前层边界 end 时，说明必须跳一步了
  • 然后把下一层边界更新成 farthest

本质上是“按层扩展”的贪心。

package main

func jump(nums []int) int {
    steps := 0
    end := 0
    farthest := 0

    for i := 0; i < len(nums)-1; i++ {
        if i+nums[i] > farthest {
            farthest = i + nums[i]
        }
        if i == end {
            steps++
            end = farthest
        }
    }

    return steps
}

面试怎么说

这题虽然看起来像搜索，但可以贪心。因为在当前这一跳能覆盖的区间里，我只需要关心下一步最远能到哪里；当走到当前区间边界时，再统一跳一步。这样可以保证跳数最少。

3. 动态规划思想

3.1 动态规划的本质

动态规划的核心不是“背公式”，而是三件事：

  1. 状态是什么
  2. 状态怎么转移
  3. 初始化是什么

很多题写成一句话就是：

dp[i] / dp[j] / dp[i][j] 表示什么，然后当前答案由之前哪些状态转移而来。

3.2 经典例题：LeetCode 70 爬楼梯

题目本质

  • 每次可以爬 1 阶或 2 阶
  • 问爬到第 n 阶有多少种方法

DP 思路

设：

  • dp[i]：爬到第 i 阶的方法数

那么：

  • 最后一步如果从 i-1 上来，有 dp[i-1] 种
  • 最后一步如果从 i-2 上来，有 dp[i-2] 种

所以：

1

dp[i] = dp[i-1] + dp[i-2]

package main

func climbStairs(n int) int {
    if n <= 2 {
        return n
    }

    dp := make([]int, n+1)
    dp[1] = 1
    dp[2] = 2

    for i := 3; i <= n; i++ {
        dp[i] = dp[i-1] + dp[i-2]
    }

    return dp[n]
}

面试怎么说

这题是最经典的一维 DP。定义 dp[i] 表示到第 i 阶的方法数，转移来自前一步和前两步，所以 dp[i] = dp[i-1] + dp[i-2]。

3.3 进阶例题：LeetCode 198 打家劫舍

题目本质

  • 一排房子，每个房子有金额
  • 不能偷相邻的两家
  • 问最多能偷多少钱

DP 思路

设：

  • dp[i]：考虑前 i+1 个房子时，最多能偷到的钱

对第 i 个房子，有两种选择：

  • 不偷它：dp[i-1]
  • 偷它：dp[i-2] + nums[i]

所以：

1

dp[i] = max(dp[i-1], dp[i-2] + nums[i])

package main

func rob(nums []int) int {
    n := len(nums)
    if n == 0 {
        return 0
    }
    if n == 1 {
        return nums[0]
    }

    dp := make([]int, n)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])

    for i := 2; i < n; i++ {
        dp[i] = max(dp[i-1], dp[i-2]+nums[i])
    }

    return dp[n-1]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

package main

func rob(nums []int) int {
    f0, f1 := 0, 0
    for _, num := range nums {
        f0, f1 = f1, max(f0+num, f1)
    }
    return f1
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

这个写法怎么理解？

  • f0 表示“处理到上一间房子时，不偷上一间 的最大收益”
  • f1 表示“处理到上一间房子时，考虑完上一间后的最大收益”

更容易记的方式是：

  • 新的 f0 = 旧的 f1
  • 新的 f1 = max(旧的 f0 + num, 旧的 f1)

含义是：

  • 如果偷当前这间，那上一间就不能偷，所以收益是 旧的 f0 + num
  • 如果不偷当前这间，那收益就是 旧的 f1

因此每次循环都在做“偷当前”还是“不偷当前”的决策。

这里最关键的一点是：

1

f0, f1 = f1, max(f0+num, f1)

Go 的这一句是同时赋值，右边会先用旧值算完，再一起赋给左边，所以不会出现“前面更新了 f0，后面算 f1 时把旧值冲掉”的问题。

面试怎么说

这题的关键是“选当前”还是“不选当前”。所以动态规划状态很好设计：dp[i] = max(dp[i-1], dp[i-2] + nums[i])。本质上就是枚举当前决策，然后从更小状态转移。

4. 背包问题

4.1 什么是 01 背包？

  • 每件物品最多选一次
  • 要么选，要么不选

典型特征：

  • “每个数只能用一次”
  • “每件物品只能选一次”

4.2 什么是完全背包？

  • 每件物品可以选无限次

典型特征：

  • “每个硬币可以重复使用”
  • “每种物品数量无限”

4.3 01 背包和完全背包的根本区别

不是公式不同，而是：

同一件物品能不能在同一轮被重复使用

所以循环顺序是背包题最关键的标志：

  • 01 背包：容量倒序
  • 完全背包：容量正序

4.4 为什么背包题通常不用贪心？

因为背包题，尤其是 01 背包，通常不满足贪心选择性质。

比如：

  • 容量 50
  • 物品：
    • w=10, v=60
    • w=20, v=100
    • w=30, v=120

如果按价值密度贪心：

  • 先选 10,60
  • 再选 20,100
  • 得到 160

但最优解其实是：

  • 20,100 + 30,120
  • 得到 220

所以：

01 背包一般不能贪心，通常要用 DP。

4.5 经典例题：LeetCode 416 分割等和子集（01 背包）

题目本质

数组能否分成两个和相等的子集？

设总和为 sum：

  • 如果 sum 是奇数，不可能
  • 如果 sum 是偶数，问题变成：

能不能从数组里选一些数，和恰好等于 sum / 2

这就是 01 背包。

DP 思路

设：

  • dp[j]：是否能凑出和 j

转移：

1

dp[j] = dp[j] || dp[j-num]

因为每个数只能选一次，所以：

  • j 必须倒序

package main

func canPartition(nums []int) bool {
    sum := 0
    for _, x := range nums {
        sum += x
    }
    if sum%2 == 1 {
        return false
    }

    target := sum / 2
    dp := make([]bool, target+1)
    dp[0] = true

    for _, num := range nums {
        for j := target; j >= num; j-- {
            dp[j] = dp[j] || dp[j-num]
            // if dp[j-num] {
                  dp[j]=true  
                }
            //
        }
    }

    return dp[target]
}

面试怎么说

这题本质是 01 背包。每个数只能选一次，目标是看能不能凑出 sum/2。状态是 dp[j] 表示是否能凑出和 j，转移是 dp[j] = dp[j] || dp[j-num]，因为每个数只能用一次，所以容量必须倒序遍历。

4.6 进阶例题：LeetCode 322 零钱兑换（完全背包）

题目本质

  • 给你若干硬币
  • 每种硬币能无限使用
  • 问凑出 amount 的最少硬币数

这就是完全背包。

DP 思路

设：

  • dp[j]：凑出金额 j 的最少硬币数

转移：

1

dp[j] = min(dp[j], dp[j-coin] + 1)

因为硬币可以重复使用，所以：

  • j 必须正序

package main

func coinChange(coins []int, amount int) int {
    inf := amount + 1
    dp := make([]int, amount+1)
    for i := 1; i <= amount; i++ {
        dp[i] = inf
    }

    for _, coin := range coins {
        for j := coin; j <= amount; j++ {
            if dp[j-coin]+1 < dp[j] {
                dp[j] = dp[j-coin] + 1
            }
        }
    }

    if dp[amount] == inf {
        return -1
    }
    return dp[amount]
}

面试怎么说

这题是完全背包，因为每种硬币可以重复用。状态是 dp[j] 表示凑出金额 j 的最少硬币数，转移是 dp[j] = min(dp[j], dp[j-coin] + 1)。因为硬币能重复使用，所以容量必须正序遍历。

5. 最后总结：面试时怎么快速判断

5.1 如果你觉得能一步一步直接选最优

先想贪心，但要问自己：

我能证明局部最优一定推出全局最优吗？

如果证明不了，就别硬贪。

5.2 如果你发现当前答案依赖前面的状态

优先往动态规划上想：

  • 状态是什么
  • 转移是什么
  • 初始化是什么

5.3 如果题目里出现“容量 / 选择 / 使用次数”

优先往背包模型上靠：

  • 每个物品只能一次 -> 01 背包
  • 每个物品无限次 -> 完全背包

5.4 最后一句口诀

贪心看“局部最优能不能推出全局最优”；
动态规划看“状态怎么定义、怎么转移”；
背包先看“物品能不能重复选”，再决定容量正序还是倒序。
